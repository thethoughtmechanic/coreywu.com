
# Digital Garden Portfolio - Documentation

## Project Overview

This is Corey Wu's digital garden portfolio - a personal space for exploring AI alignment, strategic futurism, and human-technology intersection through thoughts, experiments, and timeline content.

## Vision & Philosophy

**Core Identity**: Designer and strategic futurist focused on AI alignment and human adaptation
**Aesthetic**: Warm, earthy, approachable - like a digital garden that feels lived-in and thoughtful
**Tone**: Simple, everyday language that makes complex ideas accessible
**Purpose**: A space for authentic exploration of technology's impact on humanity

## Design Language

- **Color Palette**: Cream, warm browns, soft blacks with accent gradients
- **Typography**: Clean, readable fonts with generous spacing
- **Layout**: Card-based, mobile-first, generous whitespace
- **Interaction**: Subtle animations, gentle transitions, touch-friendly
- **Content**: Bite-sized thoughts, longer-form POVs, experimental scenarios

See [Design System Documentation](./design-system.md) for detailed specifications.

## Content Strategy

### Thought Categories
- **Thought Bite**: 2-3 min quick insights
- **POV**: 4-8 min deeper perspectives  
- **Scenario**: Future-focused explorations
- **Article**: Longer-form content

### Topic Areas
- **AI & Tech**: Where AI, code, and digital systems are central
- **Society & Power**: Structures, inequality, governance, institutions
- **Identity & Meaning**: Personal, cultural, and existential questions
- **Futures & Experiments**: Speculation, prototyping, long-term imagination

## Architecture Overview

- **Frontend**: React + TypeScript + Vite
- **Backend**: Express.js + TypeScript
- **UI**: Shadcn/ui + Tailwind CSS
- **Data**: In-memory storage with structured schemas
- **Deployment**: Replit hosting

## Development Principles

1. **Mobile-first**: Every feature should work beautifully on mobile
2. **Accessibility**: Semantic HTML, keyboard navigation, screen reader support
3. **Performance**: Fast loading, smooth animations, optimized images
4. **Simplicity**: Clean code, clear naming, documented patterns
5. **Iteration**: Build, test, refine based on real usage
