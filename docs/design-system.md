
# Design System Documentation

## Color Palette

### Primary Colors
- **Cream**: `#FBF7F0` - Primary background, warm and inviting
- **Warm Brown**: `#8B4513` - Primary text, headings, navigation
- **Soft Black**: `#2C2C2C` - Body text, secondary elements

### Accent Colors  
- **Light Brown**: `#D2B48C` - Subtle backgrounds, borders
- **Muted Grey**: `#666666` - Secondary text, metadata

### Gradient Colors
- **Teal**: `from-teal-600 to-teal-400` - Thought Bites
- **Purple**: `from-purple-600 to-purple-400` - POVs, general content
- **Orange**: `from-orange-600 to-orange-400` - Scenarios
- **Green**: `from-green-600 to-green-400` - Shipped experiments

## Typography

### Font Hierarchy
- **Headings**: Clean sans-serif, multiple weights available
- **Body**: Readable serif for longer content, sans-serif for UI
- **Code**: Monospace for technical content

### Scale
- **Mobile**: Smaller base sizes, tighter line heights
- **Desktop**: Larger base sizes, more generous spacing

## Layout Patterns

### Card Design
- **Rounded corners**: `rounded-xl` for primary cards
- **Padding**: Generous internal spacing (`p-6`, `p-8`)
- **Shadows**: Subtle elevation for depth
- **Gradients**: Color-coded by content type

### Spacing System
- **Base unit**: 4px grid system
- **Generous whitespace**: Breathing room between elements
- **Mobile optimization**: Touch-friendly spacing (44px minimum)

### Grid System
- **Mobile**: Single column, stacked layout
- **Tablet**: 2-column where appropriate
- **Desktop**: 3-column maximum, left-aligned content

## Component Patterns

### Navigation
- **Desktop**: Horizontal header with clear hierarchy
- **Mobile**: Hamburger menu with slide-out drawer
- **Active states**: Clear visual feedback
- **Accessibility**: Keyboard navigation, screen reader support

### Content Cards
- **Thought Cards**: Gradient backgrounds, clear typography
- **Experiment Cards**: Status indicators, technology tags
- **Timeline Items**: Chronological layout with visual markers

### Interactive Elements
- **Buttons**: Subtle hover states, clear touch targets
- **Links**: Underlined in content, styled in navigation
- **Forms**: Clean inputs with proper labels and validation

## Animation Guidelines

### Principles
- **Subtle**: Enhance UX without being distracting
- **Fast**: Quick transitions (200-300ms)
- **Natural**: Easing curves that feel organic
- **Purposeful**: Every animation serves a function

### Common Patterns
- **Hover effects**: Gentle color/scale changes
- **Page transitions**: Smooth content swapping
- **Loading states**: Skeleton screens, fade-ins
- **Micro-interactions**: Button presses, form feedback

## Responsive Behavior

### Breakpoints
- **Mobile**: < 768px
- **Tablet**: 768px - 1024px  
- **Desktop**: > 1024px

### Mobile-First Approach
1. Design for mobile first
2. Enhance for larger screens
3. Touch-friendly interactions
4. Readable text sizes
5. Accessible color contrasts

## Content Guidelines

### Voice & Tone
- **Conversational**: Simple, everyday language
- **Thoughtful**: Considered perspectives on complex topics
- **Accessible**: Avoid jargon, explain technical concepts
- **Personal**: Authentic voice, not corporate speak

### Content Structure
- **Scannable**: Headers, bullet points, short paragraphs
- **Contextual**: Provide background for complex ideas
- **Actionable**: Clear next steps or takeaways
- **Tagged**: Consistent categorization system
